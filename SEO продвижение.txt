Отличный вопрос — без базовой SEO-подготовки и сайт, и листинг аддона теряют «бесплатный» трафик. Я сделал компактный план + дал готовые «копи-паст» вставки для сайта и листинга, а ещё собрал ключевые фразы RU/EN.

# Что важно (коротко)

* **`meta keywords` не нужны и даже вредны** (Google их не использует; лучше не плодить «спам-сигнал») ([Official Google Webmaster Central Blog][1])
* **Разметь локализацию RU/EN через `hreflang` (+ `x-default`)** — это основа для двух языков. Эту схему понимают Google и Яндекс. ([Google for Developers][2])
* **FAQ размечай JSON-LD `FAQPage`** — даёт шанс на расширенные сниппеты. Используй `FAQPage` (а не `QAPage`). ([Google for Developers][3])
* **AMO-листинг:** теги, заголовок, саммари и ключи — основа находчивости в каталоге. ([Firefox Extension Workshop][4])

# 2) Сайт: готовая «голова» для всех страниц (RU/EN)

ВСТАВЬ в `<head>` каждой страницы СРАЗУ ПОСЛЕ `<meta charset="utf-8">` и подставь `{{...}}`.

```html
<!-- Canonical -->
<link rel="canonical" href="{{CANONICAL_URL}}" />

<!-- Hreflang RU/EN (+ x-default) -->
<link rel="alternate" href="https://lifeundo.ru{{PATH_RU}}" hreflang="ru" />
<link rel="alternate" href="https://lifeundo.ru{{PATH_EN}}" hreflang="en" />
<link rel="alternate" href="https://lifeundo.ru{{PATH_EN}}" hreflang="x-default" />

<!-- Title / Description -->
<title>{{PAGE_TITLE}}</title>
<meta name="description" content="{{PAGE_DESCRIPTION}}">

<!-- Open Graph / Twitter -->
<meta property="og:type" content="website">
<meta property="og:url" content="{{CANONICAL_URL}}">
<meta property="og:title" content="{{PAGE_TITLE}}">
<meta property="og:description" content="{{PAGE_DESCRIPTION}}">
<meta property="og:image" content="https://lifeundo.ru/assets/logo.png">
<meta name="twitter:card" content="summary_large_image">

<!-- Icons / theming -->
<link rel="icon" href="/assets/icon32.png">
<meta name="theme-color" content="#0b1220">

<!-- JSON-LD Organization + WebSite -->
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "LifeUndo",
  "url": "https://lifeundo.ru/",
  "logo": "https://lifeundo.ru/assets/logo.png",
  "sameAs": ["https://t.me/LifeUndo","https://vk.com/lifeundo"]
}
</script>
<script type="application/ld+json">
{
  "@context":"https://schema.org",
  "@type":"WebSite",
  "url":"https://lifeundo.ru/",
  "name":"LifeUndo",
  "inLanguage":"{{ru|en}}",
  "potentialAction":{
    "@type":"SearchAction",
    "target":"https://lifeundo.ru/search?q={query}",
    "query-input":"required name=query"
  }
}
</script>
```

`hreflang`/`x-default` — именно так и надо для RU/EN. ([Google for Developers][2])

# 3) Разметка FAQ (только на /faq/ru.html и /faq/en.html)

В самый низ страницы, перед `</body>`:

```html
<script type="application/ld+json">
{
 "@context":"https://schema.org",
 "@type":"FAQPage",
 "mainEntity":[
  {"@type":"Question","name":"Что делает LifeUndo?",
   "acceptedAnswer":{"@type":"Answer","text":"Восстанавливает недавние вводы текста, закрытые вкладки и историю буфера. Все данные локальны."}},
  {"@type":"Question","name":"Где хранятся мои данные?",
   "acceptedAnswer":{"@type":"Answer","text":"Только в браузере, в storage.local. Сетевых запросов и аналитики нет."}},
  {"@type":"Question","name":"Работает ли на приватных вкладках?",
   "acceptedAnswer":{"@type":"Answer","text":"Да, если дать разрешение «Запуск в приватных окнах»."}},
  {"@type":"Question","name":"Поддерживается Android/Chrome?",
   "acceptedAnswer":{"@type":"Answer","text":"Firefox Desktop — полностью. Chrome/Edge — в работе; мобильные версии рассматриваем."}},
  {"@type":"Question","name":"Чем LifeUndo отличается от Lazarus/Typio?",
   "acceptedAnswer":{"@type":"Answer","text":"LifeUndo современный, приватный и офлайн. Альтернативы Lazarus/Typio часто заброшены."}}
 ]
}
</script>
```
